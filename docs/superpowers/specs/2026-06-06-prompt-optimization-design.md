# CodePilot 提示词与工具策略设计

本文档描述当前提示词结构、工具选择约束和评测要求。

## 提示词目标

CodePilot 的系统提示需要同时满足四件事：

1. 明确 Agent 身份和任务边界。
2. 引导模型优先使用专用工具读取、搜索和编辑项目。
3. 在长任务中保持计划、执行、验证的节奏。
4. 保持中文用户输入的语言一致性和足够详细的反馈。

## 提示词来源

| 来源 | 作用 | 关键模块 |
|------|------|----------|
| 基础系统提示 | 通用工作原则、工具使用原则、输出要求 | `codepilot/agent/prompts.py` |
| Agent 提示 | 按 `build`、`plan`、`plan-execute`、`explore`、`general` 调整行为 | `codepilot/agent/registry.py` |
| 项目指令 | 读取仓库内 `AGENTS.md`、`CLAUDE.md`、`.claude/` | `codepilot/project/` |
| Skills | 按需注入专门技能说明 | `codepilot/skills/` |
| MCP 工具描述 | 将外部 MCP Server 能力暴露给模型 | `codepilot/mcp/` |
| 上下文提示 | 已读文件、压缩摘要、工具预算、剩余步骤 | `codepilot/agent/nodes.py` |

## Agent 行为要求

### build

- 适合默认开发任务。
- 先理解任务和相关文件，再编辑。
- 修改后优先运行项目已有测试或最小验证命令。
- 回复中说明改了什么、如何验证、仍有哪些风险。

### plan

- 适合只读分析、设计方案、任务拆解。
- 可以读取文件、搜索项目、查看 Git 信息。
- 不执行写入、编辑、危险命令或子任务。

### plan-execute

- 适合多步骤开发任务。
- 先生成明确计划，再执行工具调用。
- 执行中可根据新证据调整计划，但最终需要总结计划完成情况。

### explore

- 适合子 Agent 独立探索。
- 输出应包含文件路径、关键发现和下一步建议。
- 不修改项目。

### general

- 适合通用子任务、解释、草拟文档。
- 可在主 Agent 委派时独立运行，结果回传给主对话。

## 工具选择规则

| 任务 | 首选工具 | 避免 |
|------|----------|------|
| 查看文件内容 | `read_file` | 用 Shell 拼接 `cat` |
| 按文件名找文件 | `glob` | 用 Shell 拼接 `find` |
| 搜索符号或文本 | `grep` | 用 Shell 拼接 `grep` |
| 编辑已有文件 | `edit_file` | 整体重写无关内容 |
| 新建文件 | `write_file` | 先创建空文件再二次编辑 |
| 查看 Git 状态 | `git_status` / `git_diff` | 手写复杂 Git 管道 |
| 运行测试 | `run_shell` | 跳过验证直接声称完成 |
| 独立探索 | `task` + `explore` | 在主上下文中展开大量无关读取 |

## Shell 使用原则

`run_shell` 用于测试、构建、项目脚本、Git 操作和必要的系统命令。读取、搜索、编辑项目文件时，应优先使用专用工具，以便工具结果能被上下文管理器正确追踪和压缩。

## 上下文策略

- 文件读取结果会进入 `files_context`，提示模型避免重复读取相同内容。
- 大工具结果按上下文窗口自适应截断，并将完整内容保存到 truncation store。
- 消息较多时先 pruning，再 compaction，极端情况下 overflow 降级。
- 压缩摘要必须保留任务目标、已修改文件、关键命令结果和未完成事项。

## 用户体验要求

- 长任务期间持续输出动态状态。
- 需要确认时使用可选择菜单，支持方向键和回车。
- 最终回复必须包含高信号内容，避免只说“已完成”。
- 任务被阻断时说明原因、影响和可执行的下一步。

## 评测样例

| 场景 | 期望行为 |
|------|----------|
| 搜索类任务 | 使用 `grep` 或 `glob` 定位，再读取少量相关文件 |
| 文件编辑任务 | 读取目标文件，使用 `edit_file` 修改，运行最小验证 |
| 项目分析任务 | 先梳理目录，再抽样核心模块，输出结构化结论 |
| 长任务 | 使用计划、阶段反馈和最终验证，避免提前结束 |
| 权限确认 | 用户允许后继续执行，不再重复阻断同类操作 |
