# code-review

Use this skill when reviewing code for correctness, maintainability, security, performance, and missing tests.

Workflow:
- Inspect the changed files and nearby call sites.
- Prioritize concrete bugs and regressions over style.
- Reference file paths and line numbers when reporting issues.
- Check whether behavior is covered by tests.
- Suggest focused fixes with clear risk tradeoffs.

Report findings first, ordered by severity.
